CNAMER

713 Domains → 584,345 redirects

Redirect a domain using only a DNS record, no forms, no accounts, no database, no charge, just cnamer. A variety of options are available, switch protocols, specify HTTP status code, include a custom query string and more.

Redirect search.website.com to google.com:

search.website.com. CNAME google.com.cnamer.com.

Point a domain to the cnamer server using and specify options as part of the CNAME or a TXT record, cnamer will perform a DNS lookup and redirect the user to your specified destination. Throughout this documentation "cnamer.org" will be used as a placeholder for your domain, you can visit any cnamer.org example to see the redirect in action.

1. Usage

CNAME

Specify the destination domain as a subdomain of cnamer.com. A simple subdomain redirect of google.cnamer.org to google.com:

google.cnamer.org. CNAME google.com.cnamer.com.

Options can be specified as part of a cname redirect as key.value pairs that proceed an -opts- flag, after the target domain. For example to redirect wikipedia.cnamer.org to wikipedia.org with a 301 status code:

wikipedia.cnamer.org. CNAME wikipedia.org-opts-status.301.cnamer.com.

Multiple options should be separated by dashes, for example a redirect to https://wikipedia.org with a 301 status code:

wikipediassl.cnamer.org. CNAME wikipedia.org-opts-statuscode.301-protocol.https.cnamer.com.

TXT

TXT records are an alternative method of specifying redirect properties, the target and options should be encoded with JSON and set as the value of a TXT record matching your redirect domain. For example to redirect github.cnamer.org to https://github.com with a status code of 301 create a TXT:

Name github.cnamer.org
VALUE {"destination":"https://github.com/", "options": {"statuscode": 301}}

Then create a CNAME pointing your chosen domain to txt.cnamer.com, this tells cnamer to look for properties in the TXT records:

github.cnamer.org. CNAME txt.cnamer.com

A Record

A root domain (eg: cnamer.org) cannot be a CNAME, a workaround for this is supported: point the A record for the root domain to the cnamer server (CNAMER_IP) and then creating a CNAME matching the root domain (using CNAME or TXT options as described above). For example to redirect cnamer.org to cnamer.com:

A cnamer.org. IN A 176.58.124.239
CNAME cnamer.org.cnamer.org. CNAME example.org.cnamer.com

2. Options

There are a wide variety of use cases for domain redirects, each has unique requirements and cnamer provides options to allow for the flexibility most situations will need.

Option Values Description
protocol https, http, ftp Protocol for the target URL
statuscode 301, HTTP Status Code HTTP Status Code to be used in the redirect
query true or false Append query string (if any) to the target URL
querystring string Custom query string (multi values joined in order)
source true or false Include source of the request as GET param 'source'

validation

All values listed above are only suggestions and are not enforced, you are welcome to pass any values and the system will try and use the options, if they are invalid (eg: a non-existent protocol, invalid status code) your redirect may fail.

3. Widgets

Validator

To validate a redirect configuration (either CNAME or TXT) select the type from the dropdown below and enter your configuration. This widget does not perform any DNS lookups, it just validates the records configuration.

Redirect history

A recent history of redirects for a specific CNAMER, including the inbound and outbound values for each redirect. Enter the CNAMER target URL (eg: google.com.cnamer.com) or the original URL (eg: google.cnamer.org) and the last 100 redirects will be loaded.

4. Examples

CNAME

Subdomain -> Domain

exampleca.cnamer.org. CNAME google.com.cnamer.com.

Subdomain -> Domain with 302

examplecb.cnamer.org. CNAME google.com.cnamer-opts-statuscode.302.com.

Subdomain -> Domain with request query

examplecc.cnamer.org. CNAME google.com.cnamer-opts-query.true.com.

Subdomain -> Domain with custom query

examplecd.cnamer.org. CNAME google.com.cnamer-opts-querystring.redirected.com.

TXT

Subdomain -> Domain

NAME exampleta.cnamer.org. CNAME txt.cnamer.com.
VALUE example. TXT "{"destination":"http://google.com/"}"

4. Use Cases

SaaS migration

Migrating from one platform to another can leave behind dead links across the internet, moving from help.domain.com hosted by Desk to support.domain.com hosted by Zendesk with a CNAMER redirect on the previous domain is an easy way to ensure customers end up in the right place.

support.mcf.li. CNAME support.curse.com-opts-query.true-querystring.forums-querystring.22197033.cnamer.com.

Holding Domains / Retired Domains

Domains being held for future projects or from retired projects don't need to be left pointing into the abyss, redirect to a personal site or landing page.

consumed.by. CNAME samryan.co.uk.cnamer.com.
